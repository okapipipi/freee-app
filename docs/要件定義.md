# freee会計連携アプリ 要件定義書

**作成日**: 2026年2月24日
**対象システム**: 社内経費・販管費管理システム（freee会計連携）
**本番URL**: https://freee-app-ten.vercel.app

---

## 1. プロジェクト概要

### 目的
社員が経費・販管費の申請を行い、管理者がfreee会計と連携して承認・仕訳登録を行うための社内Webアプリケーション。

### 背景
- freee会計への手入力作業を削減する
- 経費申請の承認フローを電子化する
- 経営者がリアルタイムでPL・CFを把握できるダッシュボードを提供する

---

## 2. ユーザー種別と権限

| ロール | 説明 | 主な操作 |
|---|---|---|
| **社員（employee）** | 一般社員 | 経費・販管費の申請、申請履歴の確認 |
| **管理者（admin）** | 経理担当者 | 申請の承認・否認、freee同期、ユーザー管理 |
| **役員（executive）** | 経営者 | ダッシュボード閲覧、申請・承認も可能 |

---

## 3. 機能要件

### 3-1. 認証・ユーザー管理

- メールアドレス＋パスワードでログイン（BetterAuth）
- 初回ログイン時にパスワード変更を強制
- 管理者がユーザーを招待（招待メールを送信）
- ユーザーの有効化・無効化
- ロール（社員／管理者／役員）の設定
- パスワードリセット機能

### 3-2. 申請機能（社員）

#### 立替経費申請
- 対象：社員が個人で立て替えた費用
- 入力項目：利用日、金額、勘定科目、利用内容、領収書添付
- 支払期日（dueDate）の設定

#### 販管費・SGA申請
- 対象：会社カード（黒AMEX・会社AMEX・UPSIDER等）での支払い
- 入力項目：
  - 計上日（recordingMonth）：YYYY-MM-DD形式
  - 支払日（paymentMonth）：YYYY-MM-DD形式
  - コスト種別：月額費用 / 年間費用 / 単発費用
  - 金額、勘定科目、利用内容、支払方法
  - 費用計上部門（わかる場合のみ）
  - 費用発生の終了予定日（不明ボタンあり）
- 補足：単発費用以外は月末日を選択

### 3-3. 申請履歴・確認（社員）

- **販管費タブ**：計上月でグループ表示
- **立替経費タブ**：利用日でグループ表示
- 各申請のステータス確認（下書き／申請済み／承認済み／freee連携済み）

### 3-4. 承認・管理機能（管理者）

- 申請一覧の確認・承認・否認
- 承認時のfreee会計への自動仕訳登録（メモタグ「仮」付き）
- 証憑（領収書）のfreeeへの自動アップロード
- 申請の削除・修正

### 3-5. freee会計連携

- OAuth2.0でfreeeに接続
- マスタデータの同期：
  - 勘定科目
  - 取引先
  - メモタグ（「仮」「販管費振込確認用」「給与振込確認用」を自動作成）
  - 部門
- 申請承認時にfreeeへ取引を自動登録（「仮」タグ付き）
- freeeの確定取引データを1時間ごとに自動取得・キャッシュ

### 3-6. ダッシュボード（役員・管理者）

#### 表示構成
- **部門タブ**：全体 / 営業代行 / Salesforce / Corporate / AIDOG / 人材紹介
- **メインタブ**：PL（損益計算書）/ CF（資金繰り）/ ランニングコスト

#### PL（損益計算書）
| セクション | データソース | 条件 |
|---|---|---|
| **実績** | freeeDealsCache | 「仮」タグなし（freeeで確定済み） |
| **目標** | 申請データ（expenseReport） | ステータス：申請済み以上 |

- 表示形式：部門 > 勘定科目 > 取引先 の階層ツリー
- 横軸：1〜12月の月次表示
- 現在月をハイライト表示

#### CF（資金繰り）
| セクション | データソース | 基準日 |
|---|---|---|
| **実績** | freeeDealsCache | 支払期日（dueDate） |
| **目標** | 申請データ | 支払月（paymentMonth / dueDate） |

#### ランニングコスト
- 月額費用・年間費用の一覧表示
- 月額費用のサマリーカード（月計・年換算）

#### その他機能
- 「今すぐ更新」ボタン：freeeデータを即時同期
- 最終同期日時の表示
- 年度切替（前年・翌年ナビゲーション）

## 4. 非機能要件

### インフラ
| 項目 | 仕様 |
|---|---|
| ホスティング | Vercel |
| データベース | Turso（libSQL / SQLite互換） |
| ファイルストレージ | Vercel Blob（Private） |
| ORM | Drizzle ORM |
| 認証 | BetterAuth v1.4 |

### 自動処理
- freeeデータ同期：毎時0分に自動実行（Vercel Cron）
- Vercel Proプラン必要（Hobbyは毎日1回のみ）

### セキュリティ
- 全ページ認証必須
- ロールベースのアクセス制御
- ファイルはPrivateストレージ（署名付きURLで一時アクセス）
- Cronエンドポイントは`CRON_SECRET`で保護

### メール通知
- 申請提出時・承認時に通知メール送信（Resend）

## 6. 申請ステータス遷移

```
draft（下書き）
  → submitted（申請済み）
    → approved（承認済み）
      → synced_to_freee（freee連携済み）
    → rejected（否認）
```

## 7. 月次レポートスキル（Claude Code）

### 概要

月初に `claude` 上で `/monthly-report` と打つだけで、前月の予実分析レポートを自動生成しNotionに投稿するClaudeCodeスキル。

### ゴール

経営会議向けの月次予実報告書を、手動集計なしで5分以内に完成させる。

### トリガー

```
/monthly-report         ← 前月分を自動判定
/monthly-report 2026-01 ← 年月を明示指定も可
```

### データソース

| データ | 取得方法 | 詳細 |
|---|---|---|
| **実績（P/L）** | このアプリのAPI `/api/admin/dashboard` | freeeDealsCacheの「仮」タグなし取引 = freee月次P/L相当 |
| **予算** | Google Spreadsheet（gogcliで読む） | 部門ごとにタブが分かれたスプレッドシート |
| **請求書受領状況** | Google Spreadsheet（gogcliで読む） | 受領済み・未着の管理シート |

### Notionレポートの構成

```
📄 月次レポート：YYYY年MM月

## サマリー
全部門合計：目標 XX万円 / 実績 XX万円（差異 ±XX万円）

## 部門別詳細（営業代行 → Salesforce → Corporate → その他の順）

### [部門名]
部門合計：目標 XX万円 / 実績 XX万円

| 勘定科目 | 取引先 | 目標 | 実績 | 差異 |
|---|---|---|---|---|
| 広告宣伝費 | 株式会社X | 50万 | 72万 | ⚠️ +22万 |
| ...        |          |      |      |          |

🔍 差異分析（±5万円以上の項目）
- [勘定科目 / 取引先]：+22万円。[Claudeによる原因考察・コメント]
- ...

📌 前月比変化（人件費系※除く）
- 今月新規発生：[勘定科目 / 取引先 / 金額]
- 先月あったが今月なし：[勘定科目 / 取引先]

※人件費系から除外：給料手当、役員報酬、外注費

## ランニングコスト
月額固定費一覧（サービス名・金額・部門・担当）

## 請求書受領状況
| 取引先 | 請求予定額 | 受領状況 | 備考 |
|---|---|---|---|
| 株式会社Y | 30万円 | ✅ 受領済み | |
| 株式会社Z | 15万円 | ⚠️ 未着 | 先月から持ち越し |
```

### Claudeによる自動分析ルール

1. **差異分析**: 目標値との差異が **±5万円以上** の項目について原因を考察してコメント
2. **変化の洗い出し**: 前月に存在しなかった費用が今月発生した場合、**全勘定科目を対象**に列挙
3. **予算超過の考察**: ランニングコストの変動を参照しながら「なぜ超過したか」を推論
4. **給料手当の内訳表示**: 品目タグ（「インターン」「正社員」）ごとに分けて集計・前月比を表示

### 給料手当の表示ルール

freeeの「品目タグ」を使って給料手当を内訳表示する。

```
| 勘定科目  | 品目タグ | 目標  | 実績  | 差異    | 前月実績 | 前月比 |
|-----------|----------|-------|-------|---------|----------|--------|
| 給料手当  | 正社員   | XX万  | XX万  | ±XX万  | XX万     | ±XX万 |
| 給料手当  | インターン| XX万  | XX万  | ±XX万  | XX万     | ±XX万 |
| 役員報酬  | （タグなし）| XX万 | XX万  | ±XX万  | XX万     | ±XX万 |
| 外注費    | （タグなし）| XX万 | XX万  | ±XX万  | XX万     | ±XX万 |
```

- 品目タグが複数存在する場合は各タグで1行ずつ展開
- タグのない科目は従来通り取引先ごとに集計

### スキルファイルの構成

```
.claude/skills/monthly-report    ← スキル定義（Claudeへの指示）
docs/monthly-report-spec.md      ← 詳細仕様・分析ロジック・テンプレート
```

### 実行フロー

```
1. スプレッドシートから予算データを読む（gogcli）
2. このアプリのAPIから実績データを取得
3. 請求書スプレッドシートを読む（gogcli）
4. 部門ごとに予実を突き合わせて差異計算
5. 差異±5万以上 → AI分析コメント生成
6. 前月データと比較して変化を洗い出し
7. Notionに投稿（月ごとのページとして蓄積）
```

### Notion設定

- 投稿先：既存のNotionワークスペース内の月次レポート用データベース
- 1ページ = 1ヶ月分
- 毎月蓄積して振り返りができる形式
